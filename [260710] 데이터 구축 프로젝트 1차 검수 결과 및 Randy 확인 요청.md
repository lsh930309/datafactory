## 1. 검수 개요

샘플 1건은 4개 파일이 한 세트다: 문서 이미지(jpg), 문서에 적힌 값을 옮긴 정답 데이터(json), 각 값의 위치 좌표(bbox json), 문서 유형별 항목 목록(schema json). 레지스트리 v2.2와 대조하여 파일 구성, 파일 간 정합, 값의 현실성, 작성방식(수기/인쇄) 일치를 검수했다.

| 검수 범위 | 내용 |
| --- | --- |
| 금융 | 24개 유형 (표준 세트 16종, PDF만 7종, 비표준 1종) |
| 제조 | 10개 유형 (표준 세트 9종, PDF만 1종) |
| 의료, 무역 | 11개 유형 (값 상세 검수는 일부 수행) |

## 2. 검수 결과 요약

- 양호 파일 만듦새: 이미지, 정답 데이터, 위치 좌표, 항목 목록 4개 파일이 서로 어긋나는 곳 없음 (전 폴더 오류 0건). 위치 좌표는 이미지 크롭 대조로 정확성 확인.
- 충족 1차 목표(유형당 명확한 샘플 1개): 표준 세트 보유 유형은 모두 서로 다른 샘플이 최소 2개 이상 있어 충족.
- 보완 값의 현실성, 손글씨 반영, 페이지 완결성에서 아래 재작업 항목 발생.
- 누락 레지스트리상 금융과 제조 업무에 필요한 문서 15종이 산출물에 없음.

## 6. 재작업 요청 2 — 손글씨 반영 (11개 유형)

레지스트리에서 작성방식이 "수기"인 문서인데 현재 샘플은 전부 컴퓨터 활자다. 실제 업무에서는 고객이 손으로 쓰는 문서들이라 손글씨 반영이 필요하다.

| 도메인 | 대상 문서 |
| --- | --- |
| 금융 (6종) | 여신신청서, 여신거래약정서, 계좌개설신청서, 카드발급신청서, 실소유자 확인서, 투자성향 설문 |
| 의료 (5종) | 건강진단서, 의무기록 사본, 진단서·소견서, 처방전, 수술기록지 |

손글씨를 일일이 작성하는 데 어려움이 있다면 말씀 달라. 어떻게 도움을 드릴 수 있을지 같이 고민하겠다. 참고로 아래 방법이면 자동화가 가능할 것으로 보인다.

- 손글씨 폰트 오버레이: 무료 한글 손글씨 폰트(나눔손글씨 시리즈 등 50여 종)를 샘플마다 다르게 골라 기입란 값만 손글씨 폰트로 렌더. 지금 생성 방식에서 폰트만 바꾸면 되어 부담이 가장 적음
- 스캔 느낌 후처리: 약간의 기울임, 노이즈, 밝기 변화, 그림자를 이미지에 일괄 적용하는 스크립트. 이 부분은 기획 쪽(Benson)에서 만들어 제공 가능
- 서명 칸: 손글씨 폰트 또는 서명 이미지 세트를 만들어 재사용
