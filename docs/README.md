# 문서 안내

이 폴더에는 현재 DataFactory를 재구축하거나 운영할 때 필요한 핵심 지침만 남긴다.

## 핵심 문서

- `start_idea.md`: 최초 문제 정의와 목표
- `project_rough_plan.md`, `v1_development.md`: 초기 구현 방향과 v1 계획
- `registry_workbench_implementation.md`: 레지스트리/워크벤치 흐름
- `pipeline_ready_document_authoring_playbook.md`: 작업 가능 문서 authoring 운용 절차
- `pipeline_ready_bbox_schema_refinement_plan.md`: bbox/schema 보정 원칙
- `schema_stylesheet_faker_plan.md`: schema/style/faker 설계 방향
- `authoring_backup_policy.md`: authoring 데이터 손실 방지 정책
- `20260702_cleanroom_authoring_method.md`: cleanroom 문서 제작 방식
- `20260703_final_results_export_implementation_plan.md`: 최종 산출물 export 설계
- `20260703_pure_semantic_gt_export_plan.md`: metadata 없는 GT/BBox export 정책
- `objectives/20260702_manual_authoring_objective.md`: 폐기된 과거 1차 범위 기록(현재 분류/진척도 기준으로 사용 금지)

## 제외한 문서

세부 실험 로그, 문서별 장문 QA/보정 보고서, 오래된 UI/기술 실험 계획은 GitHub 커밋 대상에서 제외하고 로컬 `.bin/trash/`에 보관한다.
