# 공개 저장소 전환 시 로컬 필수 파일 안내

이 문서는 `.gitignore`로 제외된 로컬 파일 중 DataFactory의 실행 또는 현재 작업 재현에 필요한 항목을 정리한 것이다. 실제 비밀키, 원본 문서, 폰트 바이너리는 이 저장소에 추가하지 않는다.

## 결론

- 공개 저장소만 받아도 코드, 레지스트리, bbox/review JSON, authoring schema/style/faker 상태는 확인할 수 있다.
- 다만 **현재 문서의 OCR·리뷰·인페인팅·Agent evidence 생성까지 그대로 이어가려면 원본 샘플과 폰트가 별도로 필요**하다.
- Deep OCR, PaddleOCR, LaMa, 일부 cleanroom 문서 생성은 각각 별도 자격증명·런타임·이미지 asset이 있어야 동작한다.
- `outputs/`, 미리보기, OCR 중간 결과, 로그와 백업은 프로그램 시작에 필수는 아니며 원본 샘플과 구조 JSON으로 다시 만들 수 있다.

## 1. 프로그램 실행에 필요한 로컬 환경

| 경로 | 필요 수준 | 설명 및 준비 방법 |
| --- | --- | --- |
| `.venv/` 또는 `DATAFACTORY_PYTHON`으로 지정한 Python 환경 | 기본 GUI 실행 필수 | `start_datafactory_gui.command`는 기본적으로 `.venv/bin/python`을 사용한다. 저장소에 가상환경을 올리지 말고 `python3.14 -m venv .venv` 후 `pip install -e '.[dev,inpaint]'`로 재생성한다. |
| `web/node_modules/` | 웹 GUI 실행 필수, 자동 재생성 가능 | Node.js/npm이 있으면 실행 스크립트가 폴더가 없을 때 `npm install`을 수행한다. 공개 저장소에는 `web/package-lock.json`만 유지한다. |
| `.venv-ocr/` 또는 동등한 별도 Python 환경 | OCR·PDF 변환·고급 인페인팅 사용 시 필수 | PaddleOCR, OpenCV contrib, `pypdfium2`, PyTorch, LaMa 등이 필요하다. 실행 스크립트는 `cv2`와 Pillow를 사용할 수 있는 `.venv-ocr`을 우선 선택한다. `scripts/install_lama_runtime.sh`는 기존 OCR 환경에 LaMa를 추가하는 스크립트이며 OCR 환경 전체를 처음부터 재현하는 lock 파일은 아직 없다. |
| `.cache/torch/`, `.cache/paddlex/` 등 모델 캐시 | 온라인 환경에서는 선택, 오프라인 실행 시 사실상 필수 | 모델을 다시 내려받을 수 있으면 삭제해도 된다. 폐쇄망 배포 시에는 별도 모델 번들 또는 사내 artifact 저장소가 필요하다. |

기본 실행 예시는 다음과 같다.

```bash
python3.14 -m venv .venv
source .venv/bin/activate
pip install -e '.[dev,inpaint]'
npm --prefix web ci
./start_datafactory_gui.command
```

## 2. 기능별 비밀 파일

### Deep OCR

- 경로: `.env/deep_agent_api_key`
- 필요 기능: 클린룸 샘플의 Deep OCR 호출
- 파일 형식:

```dotenv
API_KEY="발급받은 키"
WEBHOOK_SECRET="발급받은 웹훅 시크릿"
```

이 파일은 절대 커밋하지 않는다. POSIX 환경에서는 `chmod 600 .env/deep_agent_api_key`를 권장한다. Deep OCR을 사용하지 않으면 GUI의 다른 기능에는 필요하지 않다.

### Jira 보조 스크립트

- 현재 로컬 파일: `.env/jira_api_token`
- 필요 기능: `scripts/jira_inspect_issue.py`, `scripts/jira_bulk_upload.py`
- 본 프로그램과 일반 문서 제작 흐름에는 필요하지 않다.
- 환경변수 `JIRA_API_TOKEN`, `JIRA_EMAIL`, `JIRA_BASE_URL` 또는 대응하는 `.env/` 파일로 별도 제공한다.

## 3. 현재 문서 작업을 이어가는 데 필요한 데이터

| 경로 | 필요 기능 | 공개 저장소 처리 원칙 |
| --- | --- | --- |
| `workbench/documents/**/samples/` | 기존 문서 선택, 원본 표시, OCR, bbox review, 인페인팅, Agent visual evidence 재생성 | **현재 workbench 작업을 그대로 이어가려면 가장 중요한 로컬 데이터다.** 원본·개인정보·라이선스를 검토한 뒤 별도 비공개 데이터 번들로 보관한다. |
| `seed_samples/` | 신규 seed 스캔, workbench import 및 일부 문서별 cleanroom 스크립트 | 공개 저장소에는 넣지 않고 승인된 샘플만 외부 저장소에서 복원한다. 이미 import된 문서는 보통 workbench의 `samples/`가 직접 실행 원본이다. |
| `fonts/` | 한글 렌더링과 기존 스타일 재현 | 프로그램은 시스템 폰트로 fallback할 수 있지만 결과 모양과 글자 폭이 달라진다. 현재 JSON/스크립트는 주로 `gulim.ttc`, `malgun.ttf`, `batang.ttc`, `Hana2-Regular.otf`, `NanumBarunGothic.otf`, `NanumGothic.otf`를 참조하며 일부 스크립트는 `malgunbd.ttf`, `malgunsl.ttf`, `NGULIM.TTF`, `NanumGothicBold.otf`도 사용한다. 각 폰트의 재배포 라이선스를 확인하고 설치 안내 또는 별도 사내 번들로 제공한다. |
| `assets/cleanroom_generated/` | 도장·서명·로고를 사용하는 cleanroom 렌더러 | 파일이 없으면 `cleanroom_assets.py`를 사용하는 스크립트는 `FileNotFoundError`가 발생한다. 생성 원본과 라이선스를 확인해 별도 asset 번들로 제공한다. |
| `assets/logo_pool/`, `assets/map_pool/`, `assets/photo_pool/` | APP-12 상품설명서, 감정평가서 등 특정 cleanroom 문서 | 해당 문서 생성 시에만 필수다. 지도 attribution과 생성 이미지 출처 메타데이터를 함께 보관한다. |
| `handwriting/`, `workbench/handwriting_scan_*`, `workbench/**/handwriting_pipeline/` | 필기 출력물 인쇄·스캔 intake를 이어갈 때 필요 | 기본 GUI에는 필요하지 않다. 실제 스캔과 생성 PDF가 포함될 수 있으므로 별도 비공개 작업 데이터로 취급한다. |

현재 저장소 상태를 대조하면 tracked manifest가 다수의 ignored 원본 경로를 참조한다. 따라서 새 clone에서는 JSON 상태가 보여도 대응하는 `workbench/**/samples/`가 없으면 이미지 표시와 후속 OCR/인페인팅을 그대로 재현할 수 없다.

## 4. 없어도 시작할 수 있는 재생성 대상

다음 항목은 기존 결과를 즉시 열어보는 데에는 유용하지만, 프로그램 또는 새 작업 실행의 필수 입력은 아니다.

- `outputs/`: 최종·중간 산출물. 실행 시 다시 생성한다.
- `workbench/**/ocr/`, `ocr_recrop/`: OCR 중간 결과. 원본 샘플과 OCR 런타임으로 재생성한다.
- `workbench/**/inpaint/`, `cleanup/`: 인페인팅과 수동 cleanup 결과. 원본 및 저장된 정책을 이용해 재생성한다.
- `workbench/**/authoring/render_preview*/`, `live_preview/`: 미리보기 이미지.
- `workbench/**/authoring/agent_requests/**/visual_evidence/`: Agent 요청용 crop 이미지. 선택 문서의 원본과 review 상태로 재생성한다.
- `workbench/**/authoring/agent_runs/**` 중 `job.json` 이외 파일: prompt, stdout/stderr, terminal, snapshot. 새 실행에는 필요하지 않지만 없으면 과거 터미널 로그와 실행 전 snapshot은 복원할 수 없다.
- `.bin/`, `workbench/**/backups/`, `workbench/**/review/**/backups/`: 로컬 복구용 백업.
- `.playwright-browsers/`, `.playwright-cli/`, `.pytest_cache/`, `web/dist/`: 개발·테스트·빌드 산출물.

## 5. 저장소 밖의 실행 전제

아래 항목은 이 저장소의 ignored 파일은 아니지만 현재 기능 사용에 필요하므로 공개 전환 시 별도로 안내해야 한다.

- Agentic Authoring을 실행하려면 인증된 `codex` CLI가 PATH에 있어야 한다.
- PDF import에는 `pypdfium2`, 일부 cleanroom PDF 스크립트에는 PyMuPDF(`fitz`)가 필요하다.
- 필기 스캔 QR 인식 품질을 높이려면 OpenCV contrib와 선택적인 WeChat QR 모델 4종이 필요하다. 현재 모델 기본 경로는 다른 로컬 프로젝트를 가리키므로 향후 환경변수 또는 프로젝트 내부 로컬 경로로 바꾸는 것이 좋다.

## 6. 공개 전환 전에 권장하는 후속 조치

1. 원본 샘플, workbench 샘플, 폰트, cleanroom asset을 공개 저장소와 분리한 **비공개 데이터 번들**로 만든다.
2. 데이터 번들에는 파일 자체 대신 공개 가능한 manifest, 상대 경로, SHA-256, 라이선스/개인정보 등급을 기록한다.
3. 실제 값이 없는 `.env.example`을 추가하고 DeepAgent/Jira 키 발급 및 권한 설정만 문서화한다.
4. `.venv-ocr`을 재현할 수 있도록 OS별 requirements 또는 lock 파일을 만든다.
5. 민감하지 않은 최소 demo 문서·폰트·asset을 별도 라이선스로 제공해 공개 clone에서도 smoke test가 가능하게 한다.
6. 공개 전환 전 Git 전체 이력에 비밀키나 실제 개인정보 원본이 과거에 커밋된 적이 없는지 별도 secret/PII scan을 수행한다.

> 요약하면 `.gitignore` 규칙 자체는 유지하는 편이 안전하다. 공개 저장소에는 실행 방법과 빈 설정 예시만 두고, 샘플·폰트·asset·자격증명은 외부 배포 또는 사용자가 직접 준비하는 방식으로 운영한다.
