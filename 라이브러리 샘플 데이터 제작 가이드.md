# 라이브러리 샘플 데이터 제작 가이드

**용도:** 라이브러리(도메인 홈)에 노출되는 샘플 문서를 외부 데이터 제작 담당자에게 요청할 때 전달하는 데이터 양식 가이드. 제작된 데이터는 백엔드 seed 파이프라인(`scripts/seed_library_samples.py`)이 그대로 적재하고, 이후 admin 페이지에서 관리한다.

**상태:** 본 가이드의 전체 양식(이미지·값·위치·다중페이지 `page_N`·중첩 그룹·마스킹 지정 `sample_NNN-pii.json`)은 현행 백엔드가 그대로 적재 가능 (마스킹 지정 파일: [PR #102](https://github.com/KDL-Solution/DeepHub-API/pull/102), 2026-07-13 배포).

# 만들 것

문서 종류 1개당 폴더 1개를 만듭니다. 폴더에는 필드 정의 1개(schema.json)와 샘플 여러 건(권장 3건에서 5건)이 들어갑니다.

샘플 1건의 구성은 네 가지입니다.

| 파일 | 내용 | 필수 여부 |
| --- | --- | --- |
| `sample_000.jpg` | 문서 이미지 | 필수 |
| `sample_000.json` | 필드 값 | 필수 |
| `sample_000-bbox.json` | 필드 위치 | 권장 (없으면 값만 노출) |
| `sample_000-pii.json` | 마스킹 대상 키 목록 | 선택 (없으면 자동 판정만) |

# 폴더 구조 — 실제 예시 2가지

한 페이지 문서 (사업자등록증):

```
금융/ID-01_사업자등록증/
├── schema.json
├── sample_000.jpg
├── sample_000.json
├── sample_000-bbox.json
├── sample_000-pii.json
├── sample_001.jpg
├── sample_001.json
├── sample_001-bbox.json
└── ...
```

여러 페이지 문서 (법인등기부등본, 2페이지):

```
금융/ID-02_법인등기부등본(등기사항전부증명서)/
├── schema.json
├── sample_000_p01.jpg      ← 1페이지 이미지
├── sample_000_p02.jpg      ← 2페이지 이미지
├── sample_000.json         ← 값은 문서 전체에서 1개
├── sample_000-bbox.json    ← 위치도 1개 (페이지별로 나누지 않음)
└── sample_000-pii.json
```

규칙:

- 도메인 폴더는 다음 8개 중 하나입니다: 금융, 제조, 의료, 무역, 보험, 공공, 회계, 건설
- 문서 폴더명은 `문서ID_문서명` 형식입니다. 첫 밑줄 앞이 문서 ID(예: ID-01), 뒤가 화면에 표시되는 문서명입니다.
- 샘플 번호는 `sample_000`부터 시작하고, `sample_000`이 목록 화면의 대표(썸네일)가 됩니다.

# 1. schema.json — 필드 정의 (문서당 1개)

추출할 필드의 키만 나열하고 값은 전부 빈 문자열로 둡니다.

```json
{
  "발급번호": "",
  "상호(법인명)": "",
  "사업자등록번호": "",
  "대표자 성명": "",
  "법인등록번호": "",
  "사업장 소재지": "",
  "개업일": "",
  "업태": "",
  "종목": "",
  "발급일자": ""
}
```

# 2. sample\_000.json — 필드 값

```json
{
  "발급번호": "2023-11-052345",
  "상호(법인명)": "주식회사 미래솔루션",
  "사업자등록번호": "123-45-67890",
  "대표자 성명": "이민혁",
  "법인등록번호": "110111-1234567",
  "사업장 소재지": "서울특별시 강남구 테헤란로 123, 4층",
  "개업일": "2015.03.02",
  "업태": "제조업·도매업·서비스업",
  "종목": "컴퓨터 및 주변장치 제조업",
  "발급일자": "2023.11.29"
}
```

규칙:

- 키는 schema.json과 완전히 동일해야 합니다.
- 값은 이미지에 실제 인쇄된 표기 그대로 적습니다 (띄어쓰기, 마침표, 단위 포함).
- 이미지에 해당 항목이 없으면 빈 문자열로 둡니다.
- 인물명, 기업명, 각종 번호는 전부 가상으로 만듭니다. 실존 개인이나 기업의 정보는 사용할 수 없습니다. 문서 양식 자체는 실제 서식을 참고해도 됩니다.

# 3. sample\_000-bbox.json — 필드 위치

한 페이지 문서는 필드를 바로 넣습니다.

```json
{
  "발급번호":     { "l": 0.075, "t": 0.14,  "r": 0.235, "b": 0.166 },
  "상호(법인명)":  { "l": 0.3,   "t": 0.18,  "r": 0.52,  "b": 0.206 },
  "사업자등록번호": { "l": 0.3,   "t": 0.208, "r": 0.49,  "b": 0.233 }
}
```

여러 페이지 문서는 최상위를 `page_1`, `page_2` 키로 감싸고, 각 필드를 그 필드가 실제로 보이는 페이지 아래에 넣습니다.

```json
{
  "page_1": {
    "법인등록번호": { "l": 0.85, "t": 0.105, "r": 0.965, "b": 0.123 },
    "목적":       { "l": 0.57, "t": 0.243, "r": 0.72,  "b": 0.328 }
  },
  "page_2": {
    "임원":       { "l": 0.63, "t": 0.443, "r": 0.89,  "b": 0.465 },
    "자본금":      { "l": 0.63, "t": 0.73,  "r": 0.85,  "b": 0.752 }
  }
}
```

규칙:

- 각 필드 값이 이미지에서 차지하는 사각형입니다. l은 왼쪽, t는 위, r은 오른쪽, b는 아래 경계입니다.
- 좌표는 픽셀이 아니라 해당 페이지 이미지 크기 대비 비율(0에서 1)입니다. 픽셀 좌표를 이미지 너비(l, r)와 높이(t, b)로 나눈 값이며 소수 3자리에서 4자리면 충분합니다.
- 값이 빈 필드는 생략해도 됩니다.

# 4. sample\_000-pii.json — 마스킹 대상 지정

개인정보로 취급해 마스킹할 필드의 키 목록입니다.

```json
["대표자 성명", "법인등록번호", "사업장 소재지"]
```

규칙:

- 키 표기는 sample\_000.json과 완전히 동일해야 합니다.
- 여러 페이지 문서도 파일은 1개입니다 (키 단위 지정이라 페이지 구분이 없습니다).
- 파일을 생략하면 시스템의 자동 판정(주민등록번호, 연락처, 주소 같은 키 이름 기준과 이미지 검출)만 적용됩니다.

# 5. 중첩 그룹(하위 항목)이 있는 문서

필드를 그룹으로 묶은 중첩 구조를 그대로 사용할 수 있습니다. 펼쳐 쓸(flatten) 필요가 없고 중첩 깊이 제한도 없습니다. 화면에도 그룹 계층 그대로 표시됩니다. 예시 (선하증권 축약):

```json
{
  "문서정보": {
    "선하증권번호": "",
    "원본발행부수": ""
  },
  "당사자": {
    "송하인": { "상호": "", "주소": "" },
    "수하인": { "상호": "", "주소": "", "전화": "" }
  },
  "운송정보": {
    "선박항차": "",
    "선적항": ""
  }
}
```

규칙:

- 값 JSON과 위치 JSON도 schema.json과 **동일한 중첩 구조**로 만듭니다. 위치 JSON은 좌표를 넣을 최하위 필드만 같은 경로 아래에 두면 됩니다 (예: `{"당사자": {"송하인": {"상호": {"l": ..., "t": ..., "r": ..., "b": ...}}}}`).
- 여러 페이지 문서는 `page_N`이 최상위이고 그 아래에 중첩 구조를 둡니다.
- 마스킹 파일(sample\_NNN-pii.json)은 **최하위 키 이름 단위**이며 경로를 쓰지 않습니다. 같은 이름의 키가 여러 그룹에 있으면(예: 송하인의 상호, 수하인의 상호) 함께 마스킹됩니다. 그룹별로 다르게 지정해야 하면 키 이름을 다르게 만듭니다 (예: "송하인 상호", "수하인 상호").

# 6. 표처럼 반복되는 행이 있는 문서

키 뒤에 `[0]`, `[1]` 인덱스를 붙입니다. schema.json에 행 개수만큼 미리 정의하고, 값이 없는 행은 빈 문자열로 둡니다. 실제 예시 (주주명부):

```json
{
  "주주명[0]": "윤서진",
  "소유주식수[0]": "2,500",
  "주주명[1]": "",
  "소유주식수[1]": ""
}
```

마스킹도 인덱스 키를 각각 나열합니다: `["주주명[0]", "주주명[1]"]`

# 7. 이미지 규격

- 형식: JPG
- 해상도: A4 세로 기준 약 1086 x 1448 px. 장변 1400 px 이상 권장
- 용량: 페이지당 500KB 이하 권장
- 실제 스캔 문서처럼 도장, 서명, 표가 들어가면 좋습니다.

# 제출 전 체크리스트

* [ ] schema.json, 값 JSON, 위치 JSON, 마스킹 JSON의 키가 서로 일치한다 (중첩 문서는 구조까지 동일)
* [ ] 값 JSON의 내용이 이미지에 인쇄된 표기와 일치한다
* [ ] 좌표가 전부 0에서 1 사이이고, l보다 r이 크고 t보다 b가 크다
* [ ] 여러 페이지 문서는 이미지 파일명이 \_p01, \_p02 순서이고 위치 JSON이 page\_N으로 감싸져 있다
* [ ] 개인정보가 전부 가상이다
* [ ] JSON이 UTF-8이고 파싱 에러가 없다
